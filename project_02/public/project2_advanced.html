프로젝트 2 — OLED Deposition 데이터 시각화 & HTML 보고서

1. 프로젝트 개요

OLED deposition 공정 데이터를 읽고 그래프와 통계 분석으로 공정 인사이트를 찾은 뒤 HTML 보고서로 문서화하는 프로젝트입니다. 상 난이도에서는 Quarto 문서(.qmd) 하나로 코드, 그래프, 통계표, 해석, 결론이 모두 재현되는 HTML 보고서를 만듭니다.

  • 수행 기간: CH.2 이후 주말 2일 (토~일)
  • 사용 도구: Gemini AI + Python + pandas + seaborn + statsmodels + Quarto
  • 결과물: .qmd 원본 + Quarto HTML 보고서 + GitHub Repository
  • 참고 사이트: Seaborn Gallery, Quarto Docs
P1의 핵심이 "CSV를 읽고 그래프를 만든다"였다면, P2의 핵심은 "그래프를 근거로 원인 후보를 좁히고 보고서로 설명한다"입니다. 상 난이도는 여기서 한 걸음 더 나아가 재현 가능한 분석 보고서를 만드는 것이 목표입니다.

6. 단계별 수행 방법

Step 1. 환경 준비

Python 패키지와 Quarto를 설치합니다.

pip install pandas seaborn matplotlib statsmodels scipy jupyter
# Quarto 설치: https://quarto.org/docs/get-started/
quarto check

Step 2. 프로젝트 구조 설계

GitHub 저장소를 기준으로 프로젝트 폴더 구조를 설계합니다.

oled-deposition-analysis/
  oled_deposition_report.qmd    # Quarto 문서
  oled_deposition_xymap.csv     # 데이터
  images/                        # 생성된 그래프
  README.md                      # 프로젝트 설명
  requirements.txt               # Python 패키지 목록

Step 3. 데이터 탐색 및 EDA

CSV를 로드하고 기초 통계, 결측치, 데이터 분포를 확인합니다. 이 단계의 코드와 결과를 .qmd 첫 섹션에 포함하세요.

Step 4. 시각화 6종 이상 생성

아래 그래프를 최소 6종 이상 .qmd에 포함합니다:

  1. boxplot — chamber별/zone별 thickness_error_nm 분포
  2. scatter plot — 공정 온도/압력 vs yield_score
  3. heatmap (x/y map) — panel별 위치 기반 thickness_error 패턴
  4. Pareto chart — defect_type별 빈도 및 누적비율
  5. correlation heatmap — 수치형 컬럼 상관관계
  6. 추가 그래프 — 자유 선택 (pairplot, violin plot, facet grid 등)

Step 5. 통계 분석 — OLS 회귀분석

yield_score를 종속변수로 하는 다중 회귀분석을 수행합니다. 결과표를 .qmd에 포함하고 p-value를 해석합니다.

Step 6. 원인 가설 수립

x/y map hotspot과 defect_type Pareto를 연결하여 원인 가설을 세웁니다. 통계 분석의 p-value 결과와 그래프 패턴을 근거로 사용하세요.

Step 7. 결론 및 최종 권고안

"가장 먼저 확인할 공정 조건/장비/위치" 3가지 이상을 구체적으로 제안합니다. 각 권고안에는 데이터 근거(어떤 그래프, 어떤 p-value)를 명시하세요.

Step 8. Quarto 렌더링

quarto render oled_deposition_report.qmd --to html

생성된 HTML 파일을 브라우저에서 열어 코드, 그래프, 텍스트가 모두 올바르게 표시되는지 확인합니다.

Step 9. GitHub 업로드

GitHub Repository를 생성하고 .qmd, CSV, HTML, README.md를 push합니다.

git init
git add .
git commit -m "P2: OLED deposition analysis report"
git remote add origin https://github.com/your-username/oled-deposition-analysis.git
git push -u origin main

Step 10. README.md 작성

프로젝트 제목, 분석 목적, 사용 데이터, 실행 방법, 주요 발견을 간결하게 정리합니다.

p-value 결과가 이상함

  • 결측치와 문자열 컬럼이 모델에 들어갔는지 확인하세요.
  • 독립변수끼리 너무 강하게 상관되어 있으면 회귀계수가 불안정할 수 있습니다 (다중공선성).
  • VIF(Variance Inflation Factor)를 확인해서 10 이상인 변수는 제거를 고려하세요.

statsmodels OLS에서 ValueError

  • 문자열 컬럼이 독립변수에 포함되지 않았는지 확인하세요.
  • 결측치를 dropna()로 제거한 뒤 모델에 넣으세요.

GitHub push 실패

  • CSV 파일 크기가 100MB를 초과하면 Git LFS를 사용하거나 .gitignore에 추가하세요.
  • HTML 파일이 너무 크면 그래프를 별도 이미지로 분리하세요.

12. FAQ

Q. Quarto가 꼭 필요한가요?

A. 상 난이도에서는 필수입니다. Quarto는 코드, 그래프, 텍스트를 하나의 문서에서 관리하고 재현 가능한 보고서를 만드는 도구입니다. Quarto 공식 문서에서 설치 방법을 확인하세요.

Q. Streamlit 앱을 만들어도 되나요?

A. 이번 프로젝트의 기준 산출물은 Quarto HTML 보고서입니다. Streamlit은 추가 산출물로는 좋지만 필수가 아닙니다.

Q. 그래프를 많이 넣으면 높은 점수를 받나요?

A. 6종은 최소 기준이며, 각 그래프의 선택 이유와 해석이 핵심입니다. "왜 이 그래프를 선택했고, 무엇을 발견했으며, 이것이 원인 가설에 어떻게 연결되는지"를 설명하세요.

Q. 실제 회사 데이터를 써도 되나요?

A. 공개 저장소에는 절대 올리면 안 됩니다. 제공된 합성 CSV만 사용하세요.

Q. README.md에는 무엇을 넣어야 하나요?

A. 프로젝트 제목, 분석 목적, 사용 데이터, 실행 방법(quarto render 명령어), 주요 발견 요약, 폴더 구조를 포함하세요.

Q. 원인 가설은 어느 수준까지 작성하나요?

A. "chamber B의 압력 변동이 커서 thickness 균일성이 낮다 (boxplot + regression p-value 근거)" 수준으로, 그래프와 통계 결과를 구체적으로 인용하세요. 가설 3개 이상을 제시하고 각각의 데이터 근거를 명시하세요.
