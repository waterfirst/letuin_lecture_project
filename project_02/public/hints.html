Project 2 상세 힌트

5~10강 이후 과제입니다. 목표는 Streamlit으로 데이터 시각화/공정 시뮬레이터를 만들고 Streamlit Cloud에 배포하는 것입니다.

도구 선택 기준

난이도 하

Gemini Chat + Streamlit

Gemini에게 app.py 전체를 만들게 하고, 로컬에서 streamlit run app.py로 실행합니다. API 연동 없이도 통과 가능한 기본 버전입니다.

난이도 중

Google AI Studio + Gemini API

AI 해석 기능을 넣으려면 Google AI Studio에서 API Key를 만들고, Streamlit Secrets에 저장합니다. 코드 안에 Key를 직접 쓰지 않습니다.

자료 정리

NotebookLM

공정 설명, OLED/LCD 차이, 데이터 컬럼 의미를 정리할 때 사용합니다. 앱 실행 자체에는 필수가 아닙니다.

난이도 하: API 없이 만드는 기본 앱

  1. Gemini Chat에 아래 프롬프트를 넣습니다.
  2. app.py와 requirements.txt를 만듭니다.
  3. 로컬에서 실행해 슬라이더와 그래프가 움직이는지 확인합니다.
  4. GitHub에 올리고 Streamlit Cloud에서 배포합니다.
Streamlit으로 제조 공정 시뮬레이터 app.py를 만들어줘.

조건:
- 분야: 디스플레이 또는 반도체 공정
- 라이브러리: streamlit, pandas, plotly, numpy만 사용
- 사용자가 온도, 압력, 시간 같은 공정 조건을 슬라이더로 바꾸면 예상 수율이 바뀌게 만들기
- plotly 그래프 2개 이상
- 샘플 데이터는 코드 안에서 생성
- requirements.txt 내용도 같이 알려줘

난이도 중: Gemini API 해석 기능 붙이기

  1. Google AI Studio에서 API Key를 발급합니다.
  2. 로컬 개발용으로 프로젝트 폴더에 .streamlit/secrets.toml을 만듭니다.
  3. Streamlit Cloud 배포 후 App settings → Secrets에 같은 값을 넣습니다.
  4. Gemini에게 현재 공정 조건과 그래프 요약을 넘겨 “면접 답변처럼 해석”하게 합니다.
# .streamlit/secrets.toml
GEMINI_API_KEY = "여기에_키를_넣기"

# app.py 안에서는 이렇게 읽기
import streamlit as st
api_key = st.secrets["GEMINI_API_KEY"]
Gemini API를 Streamlit app.py에 붙여줘.
입력값은 온도, 압력, 시간, 예상 수율이고,
출력은 다음 3개로 해줘:
1. 현재 조건 해석
2. 수율 개선 제안
3. 면접에서 설명할 수 있는 한 문장
Project 2에서는 GitHub Actions까지 하지 않아도 됩니다. 12강 이후 내용이므로, 여기서는 Streamlit Cloud 배포와 Secrets 관리까지만 잘하면 충분합니다.
