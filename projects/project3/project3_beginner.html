렛유인 AI + 바이브코딩 강의 | 최종 프로젝트 (2일) — 공정 모니터링 알림 시스템

1. 프로젝트 개요

이 강의에서 배운 모든 기술을 3가지 이상 조합하여 실제 첨단산업 현장에서 입사 1일차부터 바로 쓸 수 있는 AI 업무 자동화 도구를 만드는 최종 프로젝트입니다.

P1(데이터 분석기) + P2(AI 보고서)에서 쌓은 경험을 바탕으로, 자유 주제로 나만의 엔지니어링 솔루션을 완성합니다.

  • 수행 기간: 최종 프로젝트 주말 2일 (토~일)
  • 사용 도구: Gemini + Streamlit + Telegram Bot
  • 결과물: 센서 데이터 대시보드 + 임계값 초과 시 텔레그램 알림
  • 소요 시간: 3~4시간

5. 텔레그램 봇 생성 가이드 (Step by Step)

1봇 생성 - BotFather 찾기

Telegram 앱을 열고 상단 검색창에 @BotFather를 검색합니다. 파란색 체크 마크가 있는 공식 계정을 선택합니다.

[화면: Telegram 검색창에 "BotFather" 입력 → 공식 봇 선택]

2/newbot 명령어 입력

BotFather 채팅창에 /newbot을 입력하고 전송합니다.

[화면: 채팅창에 /newbot 입력 후 전송 버튼 클릭]

3봇 이름 입력

BotFather가 "Alright, a new bot. How are we going to call it?"라고 묻습니다. 원하는 이름을 입력합니다.

예시: MyProcessBot

[화면: 봇 이름 입력 → 전송]

4봇 username 입력

BotFather가 username을 묻습니다. 반드시 _bot으로 끝나야 합니다.

예시: my_process_monitor_bot

[화면: username 입력 → 전송 → 성공 메시지 수신]

5BOT_TOKEN 복사

BotFather가 "Done! Congratulations..."과 함께 토큰을 발급합니다. Use this token to access the HTTP API: 아래의 긴 문자열을 복사합니다.

예시: 7123456789:AAF1234abcd5678efgh9012ijkl3456mnop

주의: 토큰은 절대 공개하지 마세요!

6봇에게 메시지 보내기

Telegram 검색창에서 방금 만든 봇의 username을 검색하고, 채팅을 시작합니다. /start 또는 아무 메시지를 보냅니다.

[화면: 봇 검색 → 채팅 시작 → /start 입력]

7Chat ID 확인

브라우저에서 아래 URL을 엽니다 (BOT_TOKEN 부분을 실제 토큰으로 교체):

https://api.telegram.org/bot{BOT_TOKEN}/getUpdates

JSON 결과에서 "chat":{"id": 숫자} 부분의 숫자가 Chat ID입니다. 이것을 복사합니다.

[화면: 브라우저에 URL 입력 → JSON에서 chat → id 값 확인]

6. 단계별 수행 방법

1Step 1: 텔레그램 봇 만들기

위 섹션 5의 가이드를 따라 봇을 생성하고 BOT_TOKEN과 CHAT_ID를 메모장에 저장합니다.

2Step 2: Gemini에 프롬프트 붙여넣기

gemini.google.com에 접속하여 아래 프롬프트를 그대로 붙여넣으세요:

Streamlit으로 공정 모니터링 대시보드를 만들어줘.

기능:
1. 가상 센서 데이터 시뮬레이션 (온도, 압력, 두께)
   - 1초마다 새 데이터 생성 (정상 범위 + 가끔 이상치)
2. 실시간 게이지 차트 3개 (현재 온도, 압력, 두께)
3. 최근 50개 데이터 라인 차트
4. 상태 표시: 정상(🟢), 주의(🟡), 위험(🔴)
   - 온도 > 370: 위험, > 360: 주의
   - 압력 > 2.8: 위험, > 2.7: 주의
5. 위험 상태일 때 텔레그램으로 알림 전송
   - requests 라이브러리로 Telegram Bot API 직접 호출
   - 메시지: "⚠️ [위험] 온도 380°C 감지 — 즉시 점검 필요"
6. 사이드바: BOT_TOKEN, CHAT_ID 입력 필드
7. 모든 텍스트 한국어

telegram 알림은 requests 라이브러리의 POST로 구현해줘:
url = f"https://api.telegram.org/bot{token}/sendMessage"

3Step 3: 코드 저장

Gemini가 생성한 코드를 app.py 파일로 저장합니다.

4Step 4: 필요 라이브러리 설치

pip install streamlit plotly requests pandas

5Step 5: 앱 실행

streamlit run app.py

6Step 6: 사이드바에 토큰 입력

실행된 웹 앱의 사이드바에 BOT_TOKEN과 CHAT_ID를 입력합니다.

7Step 7: 모니터링 & 알림 확인

대시보드가 작동하면서 센서 데이터가 위험 수준을 넘을 때 텔레그램으로 알림이 옵니다. 알림이 오면 스크린샷을 캡처합니다.

8Step 8: GitHub에 업로드

GitHub에 새 리포지토리를 만들고 app.py와 requirements.txt를 push합니다.

git init
git add app.py requirements.txt
git commit -m "P3: 공정 모니터링 대시보드 + 텔레그램 알림"
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
git push -u origin main

12. 트러블슈팅

텔레그램 봇에서 메시지가 안 옴

  • @BotFather에서 봇을 만든 후, 봇에게 먼저 /start 또는 아무 메시지를 보내야 합니다
  • https://api.telegram.org/bot{TOKEN}/getUpdates에서 chat_id 재확인
  • TOKEN에 공백이나 줄바꿈이 포함되지 않았는지 확인

requests.post 호출 시 ConnectionError

  • 인터넷 연결 확인
  • 회사/학교 네트워크에서 Telegram API가 차단되었을 수 있음 (VPN 사용)

Streamlit 앱이 실행되지 않음

  • pip install streamlit plotly requests pandas 확인
  • Python 3.8 이상인지 확인
  • 파일 이름이 app.py인지 확인

Gemini API 호출 횟수 초과

  • 무료 티어는 분당 15회 제한
  • time.sleep(4) 등으로 호출 간격 조절
  • 데이터 전체가 아닌 요약만 API에 전송하여 호출 최소화

13. FAQ

Q. 주제를 못 정하겠어요.
A. 하 난이도는 "공정 모니터링 알림 시스템"으로 주제가 정해져 있습니다. 프롬프트를 그대로 사용하세요.
Q. P1, P2 코드를 재활용해도 되나요?
A. 적극 권장합니다! P1의 시각화 + P2의 보고서 + P3의 텔레그램을 합치면 강력한 도구가 됩니다.
Q. 텔레그램 봇이 작동하지 않아요.
A. 봇에게 먼저 메시지를 보내야 합니다. 봇이 먼저 메시지를 보낼 수 없습니다. Chat ID를 다시 확인하세요.
Q. 발표가 부담스러워요.
A. 3분이면 충분합니다. 화면을 보여주면서 "이 버튼을 누르면 이렇게 됩니다"만 해도 됩니다.
Q. 기술 3가지 조합이 어려워요.
A. 하 난이도는 2가지 조합 (Streamlit + Telegram)이면 충분합니다.

import requests

def send_telegram(token, chat_id, message):
    url = f"https://api.telegram.org/bot{token}/sendMessage"
    requests.post(url, json={
        "chat_id": chat_id,
        "text": message,
        "parse_mode": "Markdown"
    })
