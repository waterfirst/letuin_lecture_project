프로젝트 3 — 입사 1일차 비밀 무기: 나만의 엔지니어 AI 툴킷

렛유인 AI + 바이브코딩 강의 | 최종 프로젝트 (2일) — AI 공정 어드바이저 + 텔레그램 리포트

1. 프로젝트 개요

이 강의에서 배운 모든 기술을 3가지 이상 조합하여 실제 첨단산업 현장에서 입사 1일차부터 바로 쓸 수 있는 AI 업무 자동화 도구를 만드는 최종 프로젝트입니다.

P1(데이터 분석기) + P2(AI 보고서)에서 쌓은 경험을 바탕으로, 자유 주제로 나만의 엔지니어링 솔루션을 완성합니다.

  • 수행 기간: 최종 프로젝트 주말 2일 (토~일)
  • 사용 도구: Gemini + Streamlit + Telegram Bot + Gemini API
  • 결과물: 데이터 업로드 → AI 분석 → 보고서 생성 → 텔레그램 전송 자동화
  • 소요 시간: 6~8시간

3. 학습 목표 (중)

  1. Gemini API를 연동하여 데이터 패턴 분석 + 한국어 인사이트를 생성한다
  2. CSV/Excel 업로드 → 자동 시각화 → AI 분석 → 보고서 생성 파이프라인을 구축한다
  3. 분석 결과를 Markdown 보고서로 자동 생성한다
  4. 텔레그램으로 보고서 요약을 자동 전송한다
  5. 프롬프트를 직접 설계하는 역량을 기른다

4. 사용 도구

Gemini (대화)

코드 생성 및 디버깅. 중 난이도에서는 프롬프트를 직접 작성합니다.

Streamlit

Python 웹 대시보드. 멀티 페이지 또는 멀티 탭 구성을 권장합니다.

Telegram Bot

분석 결과 요약을 자동으로 전송. parse_mode="Markdown"으로 꾸밀 수 있습니다.

Gemini API (중 난이도 추가)

P2에서 사용했던 Gemini API를 다시 활용합니다. 데이터를 분석하고 한국어 인사이트를 생성하는 AI 분석 모듈을 구현합니다.

import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel("gemini-2.0-flash")
response = model.generate_content("이 데이터를 분석해줘: ...")

AI Tool 활용 비중

AI Tool 활용 (70%)
 - Gemini로 전체 앱 설계/구현
 - Gemini API로 분석/텍스트 생성
 - Telegram Bot 연동

도메인 지식 (30%)
 - 문제 정의 및 주제 선정
 - 알림 기준/임계값 설계

5. 텔레그램 봇 생성 가이드 (Step by Step)

1봇 생성 - BotFather 찾기

Telegram 앱을 열고 상단 검색창에 @BotFather를 검색합니다. 파란색 체크 마크가 있는 공식 계정을 선택합니다.

[화면: Telegram 검색창에 "BotFather" 입력 → 공식 봇 선택]

2/newbot 명령어 입력

BotFather 채팅창에 /newbot을 입력하고 전송합니다.

[화면: 채팅창에 /newbot 입력 후 전송 버튼 클릭]

3봇 이름 입력

BotFather가 이름을 물어봅니다. 원하는 이름을 입력합니다.

예시: MyAIAdvisorBot

4봇 username 입력 (반드시 _bot으로 끝남)

예시: my_ai_advisor_bot

5BOT_TOKEN 복사

BotFather가 발급한 토큰을 복사합니다. 토큰은 절대 공개하지 마세요!

6봇에게 /start 메시지 보내기

생성한 봇을 검색 → 채팅 시작 → /start 전송

7Chat ID 확인

https://api.telegram.org/bot{BOT_TOKEN}/getUpdates

JSON에서 "chat":{"id": 숫자} 값을 복사합니다.

12. 트러블슈팅

텔레그램 봇에서 메시지가 안 옴

  • @BotFather에서 봇을 만든 후, 봇에게 먼저 /start 또는 아무 메시지를 보내야 합니다
  • https://api.telegram.org/bot{TOKEN}/getUpdates에서 chat_id 재확인
  • TOKEN에 공백이나 줄바꿈이 포함되지 않았는지 확인

requests.post 호출 시 ConnectionError

  • 인터넷 연결 확인
  • 회사/학교 네트워크에서 Telegram API가 차단되었을 수 있음 (VPN 사용)

Gemini API 호출 횟수 초과

  • 무료 티어는 분당 15회 제한
  • time.sleep(4) 등으로 호출 간격 조절
  • 데이터 전체가 아닌 요약만 API에 전송하여 호출 최소화

Streamlit Cloud에서 텔레그램이 안 됨

  • Streamlit Cloud의 Secrets에 BOT_TOKEN, CHAT_ID를 등록했는지 확인
  • 코드에서 st.secrets["BOT_TOKEN"]으로 접근

멀티 페이지 Streamlit이 안 됨

  • pages/ 폴더명은 정확히 pages여야 합니다 (대소문자 주의)
  • 파일명 앞에 숫자를 붙여 순서 지정: 01_모니터링.py, 02_분석.py

13. FAQ

Q. 주제를 못 정하겠어요.
A. 가장 쉬운 방법: "내가 매일/매주 반복하는 귀찮은 일"을 떠올려보세요. 그것을 자동화하세요.
Q. P1, P2 코드를 재활용해도 되나요?
A. 적극 권장합니다! P1의 시각화 + P2의 보고서 + P3의 텔레그램을 합치면 강력한 도구가 됩니다.
Q. 텔레그램 봇이 작동하지 않아요.
A. 봇에게 먼저 메시지를 보내야 합니다. 봇이 먼저 메시지를 보낼 수 없습니다. Chat ID를 다시 확인하세요.
Q. 발표가 부담스러워요.
A. 3분이면 충분합니다. 화면을 보여주면서 "이 버튼을 누르면 이렇게 됩니다"만 해도 됩니다.
Q. 기술 3가지 조합이 어려워요.
A. 가장 쉬운 조합은 Streamlit(UI) + Gemini API(분석) + Telegram(알림)입니다. P1, P2에서 이미 2가지를 해봤으니 텔레그램만 추가하면 됩니다.
Q. 프롬프트를 어떻게 작성하나요?
A. P1에서 배운 프롬프트 엔지니어링을 활용하세요. 기능 목록을 구체적으로 나열하고, 한국어 응답을 요청하세요.

import requests

def send_telegram(token, chat_id, message):
    url = f"https://api.telegram.org/bot{token}/sendMessage"
    requests.post(url, json={
        "chat_id": chat_id,
        "text": message,
        "parse_mode": "Markdown"
    })
