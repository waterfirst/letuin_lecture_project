프로젝트 1: 반도체 CVD 박막 두께 Wafer Map 데이터 분석

가상 엘립소메터 데이터를 활용하여 데이터 정제(이상치/결측치 처리)를 수행하고, 반도체 웨이퍼의 박막 균일도 진단 및 불량 CVD 설비를 파악하는 반응형 보고서를 설계하는 과제 가이드라인입니다.

1. 프로젝트 개요

본 실습 프로젝트는 반도체 R&D 공정 개발 분야에서 핵심적으로 쓰이는 웨이퍼 박막 두께 균일도 분석 및 설비 매칭 분석을 주제로 합니다. 수강생들은 주어진 결함 데이터를 정제하고 두께 분포를 2D 좌표에 시각적으로 매핑하여 최외곽 또는 중심부 결함을 시각화하고 설비 이상 원인을 해석합니다.

사용 데이터셋 경로:
- 데이터셋 1 (오목형 이상): project_01/public/data/siox_thickness_data_1.csv (CVD_M02 설비 이상)
- 데이터셋 2 (볼록형 이상): project_01/public/data/siox_thickness_data_2.csv (CVD_M03 설비 이상)

2. 분석 대상 데이터 컬럼 정보 (ellipsometer 49점 포맷)

컬럼명 타입 의미 핵심 분석 가치
lot_id String Wafer Lot 번호 생산 타임라인 순으로 Lot별 특성 비교 그룹핑
cvd_machine String 가공 CVD 설비 번호 이상 거동을 보인 설비(CVD_M01~M03) 식별
meas_time String 두께 계측 완료 시간 공정 이상이 발생한 정확한 시간대 판정
position_x Float 웨이퍼 상의 측정 X 좌표 300mm 동심원 그리드 상의 2D Wafer Map 매핑 좌표계
position_y Float 웨이퍼 상의 측정 Y 좌표
thickness_value Float SiOx 박막 측정 두께 (nm) 이상치 감지 및 Lot별 두께 Uniformity 계산의 핵심 지표

3. 주말 2일간의 상세 수행 로드맵

Day 1 오전
문제 정의와 데이터셋 파악
  • 데이터셋 1 및 데이터셋 2 CSV 파일을 열어 정상 범위 외의 이상 수치(-999, 999.9, 0 등) 확인
  • 측정 좌표 그리드 형태(Center 0에서 R=145mm까지 concentric layout) 파악
Day 1 오후
데이터 정제 알고리즘 구현
  • 결측값(NaN)을 해당 Wafer의 정상 좌표 두께 평균치로 채우는 논리 구현
  • 극단값 아웃라이어를 마스킹하여 분석 대상에서 배제하는 논리 설계
Day 2 오전
2D 시각화 및 Uniformity 계산
  • Lot별 두께 편차 분석 공식 계산: Uniformity (%) = (Max - Min) / (2 * Mean) * 100
  • Wafer Map 동심원 시각화 (Cyan에서 Red로 변환되는 두께 그라데이션) 구현
Day 2 오후
이상 설비 진단 및 보고서 추출
  • 특정 설비의 두께 분포가 외곽 편중(Edge-High)인지 중심 편중(Center-High)인지 판정
  • 각 난이도 포맷에 맞춰 최종 산출물(`report_beginner.html` / `report_intermediate.html` / `app.py`) 완성

초급 복붙 권장 프롬프트 예시

나는 반도체 R&D 공정 엔지니어 취업준비생이야. 첨부된 'siox_thickness_data_1.csv' 데이터를 정밀 분석하고 싶어.
1. 이 데이터셋의 각 컬럼 형태와 데이터 개수를 파악해줘.
2. 데이터 중 엘립소메터 장비 에러로 추정되는 극단적 이상치(-999, 999.9, 0, 250.3 등)와 비어 있는 결측치(NaN)를 식별해줘.
3. 이상치와 결측치를 필터링하거나 중앙값(nominal 100nm)으로 대체했을 때와 적용하지 않았을 때의 Wafer Uniformity (%) 차이를 계산식과 표로 나타내줘.
4. 각 LOT별로 어떤 CVD 설비에서 문제가 터졌는지, 이상이 발생한 특정 시간대와 설비 번호를 판정해줘.
5. 분석 결과(Wafer Map, 설비 매칭 트렌드, 이상 원인, 엔지니어 개선안)가 모두 포함된 모던하고 아름다운 다크모드 단일 HTML 보고서 파일 (CSS 스타일 포함)을 완성해서 소스코드로 출력해줘.

6. 중급 난이도 상세 지침 및 기능 구현 힌트

JavaScript 파싱 중 문자열로 들어오는 이상치 혹은 undefined 데이터로 인해 통계 연산이 깨지지 않도록 예외 처리에 신경 써야 합니다.

핵심 기술 힌트

  • 문자열 숫자 파싱: CSV에서 읽어 들인 thickness_value가 유효하지 않은 문자이거나 공란일 경우 parseFloat()를 활용하여 NaN 처리한 후 보간합니다.
  • 2D 시각화 색상 연동: 박막 두께가 95nm 이하(Under spec)일 경우 한색 계열(Blue), 100nm 근방은 녹색(Normal), 115nm 이상(Over spec)일 경우 적색(Red) 그라데이션을 적용합니다.

7. 고급 난이도 상세 지침 및 재현성 검증 설계

고급 난이도의 최우선 요구사항은 재현성(Reproducibility)입니다. 즉, 단 하나의 데이터 파일에 하드코딩되지 않고 어떤 파일을 업로드하든 올바른 결함을 자동 분석해야 합니다.

재현성 자체 검증 시나리오:
1. siox_thickness_data_1.csv 로딩 -> CVD_M02 설비 이상 감지 및 외곽 두께 과잉(오목형) 현상 자동 경보 출력
2. siox_thickness_data_2.csv 로딩 -> CVD_M03 설비 이상 감지 및 중심 두께 과잉(볼록/돔형) 현상 자동 경보 출력
