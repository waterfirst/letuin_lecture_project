1. 프로젝트 개요

OLED deposition 공정 데이터를 읽고 그래프와 통계 분석으로 공정 인사이트를 찾은 뒤 HTML 보고서로 문서화하는 프로젝트입니다. P1에서 익힌 CSV 분석 경험을 바탕으로, 이번에는 Seaborn 시각화와 통계 분석을 활용해 더 보고서다운 결과물을 만듭니다.

  • 수행 기간: CH.2 이후 주말 2일 (토~일)
  • 사용 도구: Gemini AI + Python + pandas + seaborn + statsmodels
  • 결과물: CSV 분석 코드 + 그래프 이미지 + 통계 분석 포함 HTML 보고서
  • 참고 사이트: Seaborn Gallery
P1의 핵심이 "CSV를 읽고 그래프를 만든다"였다면, P2의 핵심은 "그래프를 근거로 원인 후보를 좁히고 보고서로 설명한다"입니다.

6. 단계별 수행 방법

Step 1. 환경 준비

필요한 라이브러리를 설치합니다.

pip install pandas seaborn matplotlib statsmodels scipy

Step 2. 데이터 로드 및 탐색

CSV를 로드하고 기초 통계를 확인합니다. 이 단계에서 결측치와 데이터 크기를 파악하세요.

힌트: df.info(), df.describe(), df.isnull().sum()을 실행하면 데이터 구조를 빠르게 파악할 수 있습니다.

Step 3. 시각화 1 — boxplot

chamber별, zone별 thickness_error_nm 분포를 boxplot으로 비교합니다.

힌트: seaborn.boxplot()에서 x='chamber', y='thickness_error_nm'을 설정하고, hue='zone'을 추가하면 그룹별 비교가 됩니다.

Step 4. 시각화 2 — scatter plot

공정 조건(온도, 압력)과 yield_score의 관계를 산점도로 확인합니다.

힌트: substrate_temp_c vs yield_score, pressure_mTorr vs yield_score를 그려보고 어느 변수가 더 강한 관계를 보이는지 비교하세요.

Step 5. 시각화 3 — heatmap 또는 x/y map

x_index, y_index별 thickness_error_nm 평균을 pivot_table로 만들어 heatmap을 그립니다.

힌트: panel별로 따로 그려야 패턴이 잘 보입니다. 특정 위치에 불량이 집중되면 장비 원인(source 정렬 등)을 의심할 수 있습니다.

Step 6. 시각화 4 — Pareto chart

defect_type별 발생 건수를 세고, 누적 비율 80%를 차지하는 상위 결함을 찾습니다.

힌트: value_counts()로 빈도를 구하고, cumsum()으로 누적비율을 계산합니다. 막대 그래프 + 누적 선 그래프 조합이 Pareto chart입니다.

Step 7. 시각화 5 — correlation heatmap

수치형 컬럼들의 상관계수를 heatmap으로 표현합니다.

힌트: df[numeric_cols].corr()로 상관행렬을 구하고 seaborn.heatmap(annot=True)로 그립니다.

Step 8. 통계 분석 — 회귀분석

yield_score를 종속변수로, 공정 조건 변수를 독립변수로 넣어 OLS 회귀분석을 수행합니다.

힌트: statsmodels의 OLS를 사용합니다. p-value < 0.05인 변수가 "통계적으로 유의한 인자"입니다. 단, R-squared 값도 함께 확인하여 모델 설명력을 판단하세요.

Step 9. HTML 보고서 작성

모든 그래프와 분석 결과를 하나의 HTML 파일로 정리합니다. 각 그래프 아래에 해석과 공정 개선 관점의 제안을 추가하세요.

힌트: 보고서 말미에 "개선 제안 3가지"를 반드시 포함하세요. 데이터 근거(그래프, p-value)와 연결지어 작성하면 좋은 점수를 받습니다.

Step 10. 프롬프트 기록 및 제출

Gemini에게 사용한 주요 프롬프트를 기록하고, 최종 산출물을 정리하여 제출합니다.

12. FAQ

Q. Streamlit 앱을 만들어도 되나요?

A. 이번 프로젝트의 기준 산출물은 HTML 보고서입니다. Streamlit은 추가 산출물로는 좋지만 필수는 아닙니다.

Q. 그래프를 많이 넣으면 높은 점수를 받나요?

A. 그래프 개수보다 "왜 이 그래프를 선택했고 무엇을 알게 되었는지"가 더 중요합니다. 5종은 최소 기준이며, 각 그래프의 해석이 핵심입니다.

Q. p-value가 뭔지 잘 모르겠어요.

A. Gemini에게 "p-value를 공정 엔지니어가 이해할 수 있게 해석해줘"라고 요청하세요. 간단히 말하면, p-value가 0.05보다 작으면 "이 변수가 yield에 영향을 준다고 볼 수 있다"는 뜻입니다. 단, AI 답변을 그대로 믿지 말고 그래프와 도메인 상식으로 재확인하세요.

Q. 실제 회사 데이터를 써도 되나요?

A. 공개 저장소에는 절대 올리면 안 됩니다. 제공된 합성 CSV만 사용하세요.

Q. 회귀분석 결과를 어떻게 해석하나요?

A. OLS summary에서 (1) R-squared: 모델 설명력, (2) 각 변수의 coef: 영향 방향과 크기, (3) P>|t|: p-value를 확인하세요. p-value < 0.05인 변수가 유의한 인자입니다.

Q. 개선 제안은 어떤 수준으로 작성하나요?

A. "chamber B의 압력 편차를 줄인다", "x/y map 좌측 하단 영역의 source 정렬을 점검한다"처럼 데이터 근거가 있는 구체적 제안을 작성하세요.
