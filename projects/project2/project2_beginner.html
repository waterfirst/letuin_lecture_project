프로젝트 2 — OLED Deposition 데이터 시각화 & HTML 보고서

1. 프로젝트 개요

OLED deposition 공정 데이터를 읽고 그래프와 통계 분석으로 공정 인사이트를 찾은 뒤 HTML 보고서로 문서화하는 프로젝트입니다. P1에서 익힌 CSV 분석 경험을 바탕으로, 이번에는 Seaborn 시각화를 활용해 더 보고서다운 결과물을 만듭니다.

  • 수행 기간: CH.2 이후 주말 2일 (토~일)
  • 사용 도구: Gemini AI + Python + pandas + seaborn
  • 결과물: CSV 분석 코드 + 그래프 이미지 + HTML 보고서
  • 참고 사이트: Seaborn Gallery
P1의 핵심이 "CSV를 읽고 그래프를 만든다"였다면, P2의 핵심은 "그래프를 근거로 원인 후보를 좁히고 보고서로 설명한다"입니다.

  • 분석 결과를 간단한 문서로 정리
  • 에러 발생 시 AI에게 디버깅 요청

heatmap이 너무 느림

  • 전체 데이터를 그대로 그리지 말고 pivot_table로 x/y 평균값을 만든 뒤 그리세요.
  • panel 하나를 선택해서 먼저 분석하세요.
# panel 하나만 선택
df_p1 = df[df['panel_id'] == 'P1']

# pivot_table로 heatmap용 데이터 생성
heatmap_data = df_p1.pivot_table(
    values='thickness_error_nm',
    index='y_index',
    columns='x_index',
    aggfunc='mean'
)

FileNotFoundError: CSV 파일을 찾을 수 없음

  • CSV 파일과 analysis.py가 같은 폴더에 있는지 확인하세요.
  • 또는 절대 경로를 사용하세요: pd.read_csv('/full/path/to/oled_deposition_xymap.csv')

그래프가 화면에 표시되지 않음

파일로 저장하는 plt.savefig()를 plt.show() 앞에 호출하세요. 순서가 바뀌면 빈 이미지가 저장됩니다.

12. FAQ

Q. Streamlit 앱을 만들어도 되나요?

A. 이번 프로젝트의 기준 산출물은 HTML 보고서입니다. Streamlit은 추가 산출물로는 좋지만 필수는 아닙니다.

Q. 그래프를 많이 넣으면 높은 점수를 받나요?

A. 그래프 개수보다 "왜 이 그래프를 선택했고 무엇을 알게 되었는지"가 더 중요합니다.

Q. p-value가 뭔지 잘 모르겠어요.

A. 하 난이도에서는 p-value를 몰라도 괜찮습니다. 기초 통계(평균, 표준편차)와 그래프 해석에 집중하세요.

Q. 실제 회사 데이터를 써도 되나요?

A. 공개 저장소에는 절대 올리면 안 됩니다. 회사 내부 데이터, 장비명, lot 정보, 고객 정보는 모두 제외하세요. 제공된 합성 CSV만 사용하세요.

Q. Gemini가 생성한 코드가 바로 실행되지 않아요.

A. 에러 메시지를 그대로 복사해서 Gemini에게 "이 에러를 고쳐줘"라고 요청하세요. 2~3번 반복하면 대부분 해결됩니다.

Q. heatmap 색상이 잘 안 보여요.

A. Gemini에게 "heatmap의 colormap을 'RdYlBu_r'로 바꾸고 colorbar를 추가해줘"라고 요청하세요.
