13. 트러블슈팅 / FAQ13. Troubleshooting / FAQ

트러블슈팅Troubleshooting

텔레그램 봇에서 메시지가 안 옴No message from Telegram bot
봇에게 먼저 /start 메시지를 보내세요. getUpdates에서 chat_id 재확인. TOKEN에 공백/줄바꿈 없는지 확인.Send /start first. Re-check chat_id from getUpdates. No spaces/newlines in TOKEN.
ConnectionError 발생ConnectionError
인터넷 연결 확인. 회사/학교 네트워크에서 Telegram API 차단 시 VPN 사용.Check internet. Use VPN if Telegram API is blocked.
Streamlit Cloud에서 텔레그램이 안 됨Telegram not working on Streamlit Cloud
Streamlit Cloud의 Secrets에 BOT_TOKEN, CHAT_ID 등록 후 st.secrets["BOT_TOKEN"]으로 접근.Register in Streamlit Cloud Secrets, access via st.secrets["BOT_TOKEN"].
Gemini API 호출 횟수 초과Gemini API rate limit exceeded
무료 분당 15회 제한. time.sleep(4)로 조절. 전체 대신 요약만 전송.Free tier: 15/min. Use time.sleep(4). Send summary, not full data.
멀티 페이지 Streamlit이 안 됨Multi-page Streamlit not working
pages/ 폴더명은 정확히 pages여야 합니다 (대소문자 주의). 파일명 앞에 숫자로 순서 지정.pages/ folder name must be exactly pages (case-sensitive). Prefix filenames with numbers for ordering.
GitHub Actions 워크플로우 실패GitHub Actions workflow failure
.github/workflows/ YAML 확인. Repository Settings > Secrets and variables > Actions에서 secrets 설정.Check .github/workflows/ YAML. Set secrets in Repository Settings > Secrets and variables > Actions.
