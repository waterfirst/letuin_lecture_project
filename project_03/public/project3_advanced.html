프로젝트 3 — 입사 1일차 비밀 무기: 나만의 엔지니어 AI 툴킷

렛유인 AI + 바이브코딩 강의 | 최종 프로젝트 (2일) — 스마트 팩토리 통합 대시보드 (포트폴리오 수준)

1. 프로젝트 개요

이 강의에서 배운 모든 기술을 3가지 이상 조합하여 실제 첨단산업 현장에서 입사 1일차부터 바로 쓸 수 있는 AI 업무 자동화 도구를 만드는 최종 프로젝트입니다.

P1(데이터 분석기) + P2(AI 보고서)에서 쌓은 경험을 바탕으로, 자유 주제로 나만의 엔지니어링 솔루션을 완성합니다.

  • 수행 기간: 최종 프로젝트 주말 2일 (토~일)
  • 사용 도구: Gemini API + Streamlit + Telegram Bot + GitHub + Streamlit Cloud 배포 + 자유 조합
  • 결과물: 입사 면접에서 시연할 수 있는 수준의 통합 엔지니어링 솔루션
  • 소요 시간: 10~12시간 (2일)
도전 수준: 이 난이도에서는 주제 아이디어 목록만 제공됩니다. 힌트 없이 스스로 설계하고 구현해야 합니다.

3. 학습 목표 (상)

  1. 배운 기술 3가지 이상을 조합하여 통합 솔루션을 설계한다
  2. Telegram Bot으로 이상 알림 + 일일/주간 리포트 시스템을 구현한다

  6. 이상 감지 알고리즘(Z-score, IQR 등)을 적용한다

7Python으로 테스트

import requests

def send_telegram(token, chat_id, message):
    url = f"https://api.telegram.org/bot{token}/sendMessage"
    requests.post(url, json={
        "chat_id": chat_id,
        "text": message,
        "parse_mode": "Markdown"
    })

4Step 4: 핵심 모듈 구현

Gemini에게 각 모듈을 개별적으로 요청합니다. 한 번에 전체를 요청하는 것보다 모듈 단위로 나누는 것이 효과적입니다.

  1. 데이터 생성/로드 모듈: 시뮬레이션 데이터 or CSV 업로드
  2. 시각화 모듈: 6종 이상 차트 (게이지, 라인, 히트맵, 박스플롯, SPC, 산점도 등)

  4. AI 분석 모듈: Gemini API로 데이터 패턴 분석 + 자연어 인사이트
  5. 텔레그램 모듈: 이상 알림 + 일일/주간 리포트
  6. 보고서 모듈: HTML 보고서 자동 생성 + 다운로드

6Step 6: 이상 감지 알고리즘 구현

Z-score 또는 IQR 방법을 사용하여 이상치를 탐지합니다.

# Z-score 예시
import numpy as np
z_scores = np.abs((data - data.mean()) / data.std())
anomalies = z_scores > 3

# IQR 예시
Q1 = data.quantile(0.25)
Q3 = data.quantile(0.75)
IQR = Q3 - Q1
anomalies = (data < Q1 - 1.5*IQR) | (data > Q3 + 1.5*IQR)

  • https://api.telegram.org/bot{TOKEN}/getUpdates에서 chat_id 재확인
  • TOKEN에 공백이나 줄바꿈이 포함되지 않았는지 확인

requests.post 호출 시 ConnectionError

  • 인터넷 연결 확인
  • 회사/학교 네트워크에서 Telegram API가 차단되었을 수 있음 (VPN 사용)

Streamlit Cloud에서 텔레그램이 안 됨

  • Streamlit Cloud의 Secrets에 BOT_TOKEN, CHAT_ID를 등록했는지 확인
  • 코드에서 st.secrets["BOT_TOKEN"]으로 접근

Gemini API 호출 횟수 초과

  • 무료 티어는 분당 15회 제한
  • time.sleep(4) 등으로 호출 간격 조절
  • 데이터 전체가 아닌 요약만 API에 전송하여 호출 최소화

GitHub Actions 워크플로우 실패

  • .github/workflows/ 폴더에 YAML 파일 확인
  • secrets 설정: Repository Settings → Secrets and variables → Actions

멀티 페이지 Streamlit이 안 됨

  • pages/ 폴더명은 정확히 pages여야 합니다 (대소문자 주의)
  • 파일명 앞에 숫자를 붙여 순서 지정: 01_모니터링.py, 02_분석.py

Streamlit Cloud 배포 실패

  • requirements.txt에 모든 라이브러리가 명시되어 있는지 확인
  • Python 버전 호환성 확인 (3.8~3.11 권장)
  • Secrets 설정 누락 확인

13. FAQ

Q. 주제를 못 정하겠어요.
A. 가장 쉬운 방법: "내가 매일/매주 반복하는 귀찮은 일"을 떠올려보세요. 그것을 자동화하세요.
Q. P1, P2 코드를 재활용해도 되나요?
A. 적극 권장합니다! P1의 app.py + P2의 ai_analyzer.py를 기반으로 P3를 시작하면 효율적입니다.
Q. 텔레그램 봇이 작동하지 않아요.
A. 봇에게 먼저 메시지를 보내야 합니다. 봇이 먼저 메시지를 보낼 수 없습니다. Chat ID를 다시 확인하세요.
Q. 발표가 부담스러워요.
A. 3분이면 충분합니다. 배포 URL로 시연하면 환경 설정 없이 바로 보여줄 수 있습니다.
Q. 기술 3가지 조합이 어려워요.
A. 가장 쉬운 조합은 Streamlit(UI) + Gemini API(분석) + Telegram(알림)입니다. 여기에 Plotly(시각화), GitHub(코드 관리), Streamlit Cloud(배포)를 추가하면 됩니다.
Q. 배포가 처음이에요.
A. Streamlit Cloud는 GitHub 리포지토리만 있으면 3분 안에 배포 가능합니다. share.streamlit.io에서 "New app" → 리포지토리 선택하면 됩니다.
Q. README를 영문으로 써야 하나요?
A. 영문 권장이지만 필수는 아닙니다. 영문 README는 포트폴리오 가치를 높여줍니다. Gemini에게 한국어 README를 영어로 번역해달라고 요청하면 됩니다.
